What is diceware?

Diceware (wikipedia)-based password generation is part of a method of generating passwords that was popularized by a popular XKCD comic from 2011.
The concept is simple: Generate long, secure, easily accessible passwords by sticking a few randomly chosen dictionary words together.

What does this website do for me?

Nowadays, most websites have somewhat awkward password requirements, such as:

  • At least one lowercase character
  • At least one uppercase character
  • At least one number
  • At least one symbol/special character
  • A minimum length
  • Or sometimes, a maximum length (some services don't allow more than 8-12 characters for passwords)

So, based on the XKCD comic and the basic principle of diceware, a typical random password might be:

reroute reformist vicinity

But you'll quickly note that this password does not meet password requirements for most websites! There are no numbers, uppercase characters, or special characters.

So, this website resolves this problem by taking a simple approach: Appending a number and one easy-to-type special character to the end of the password. You can customize the maximum length of the password, as well as the separator used between each word in the password (some websites may not allow spaces, for example).

How should passwords work for me?

Your passwords should, ideally, be:

  • long enough to ensure they cannot be easily brute-force guessed (32-128 characters length; most allow 64 without issue)
  • random enough to, at the very least, ensure uniqueness amongst yours and other leaked passwords
  • and easily accessible, otherwise you might give up and default to repeatedly using a small set of only a few easily memorable, short, and likely insecure passwords that your brain can remember.
The last bullet point about having easily accessible passwords is very important. This is typically the biggest challenge when it comes to password security, as it requires you to test different password storage and entry mechanisms until you find something that works.

How do I make my secure passwords easily accessible?

I recommend using a password manager, as do many modern security experts - a popular option is Bitwarden, but there are other password manager services:

  • Bitwarden
  • 1Password
You can also use a self-contained password storage utility called KeePass (or KeePassXC), which keeps all of your passwords contained in a single .kdbx file. This is a desirable method (even though it's more involved), because you can use an open source KeePass client called KeeWeb to store passwords in a file located in Google Drive, Teams, OneDrive, Dropbox, a WebDAV server, a local file, or other options. For iOS, you can use Strongbox to access your .kdbx file and enter your secure passwords.

About & Privacy

I absolutely do not store any passwords that are generated, or any data about you at all. Note that your data travels across many different parts of a global network when interacting with any site, so I cannot guarantee that your data is 100% secure.

Note that the actual password generation does not occur in your browser. The password is generated on my server and then sent to you. If this does not fit within your threat model, do not use this site. I want to re-emphasize that I do not store any generated passwords anywhere, but please use password generation software responsibly, according to your needs.

I chose to write a server-side API to generate passwords so that I could support arbitrarily large word lists without requiring users to consume a large amount of network bandwidth to simply generate a password. If I hadn't done this, it would require you to download a few megabytes of data each time you visit the site (with caching, this can be reduced to perhaps once per day or less). For individuals that are on a metered plan, this is not desirable, and it is also not desirable for me to host this data (as I get billed for network traffic). Additionally, I do not want users to make API requests to any third-party services in order to download the word lists upon visiting the site either - so this is what informed my decision to generate passwords server-side.

Word lists used by this application may differ from the linked upstream sources.

Password generation requests that take longer than 1 second will be canceled.
